Set概叙

首先它是一个接口
集合元素无序(存储顺序与取出顺序不一致),可重复
对比它的父接口Collection,并没有特有方法

Set集合唯一性(unique)验证

使用实现子类HashSet存储对象,源码分析

需要观察add(E e)方法,可以发现HashSet类的集合容器是一个map,调用map.put(e,PRESENT)其中的PRESENT也是一个静态常量
由于是用数组的索引与处理后的对象哈希值作运算存储数据 故而数据也是无序的
接下来观察map的put方法即可    jdk 8
jdk7 put方法
通过以上分析可以发现 Set的add方法如果元素的hash和地址值一致  或者equals一致 那么是添加不成功的 故而保证了唯一性

自定义类验证

如果要以对象的内容为判断重复的标准,重写Object类的hashCode与equals方法即可

重写hashCode

分析:
    根据源码;如果对象的hash值不同 首先可能就直接往hash表里面添加了
    所以首先要重写hashCode 使得hash值相同

固定hashCode返回值带来的问题

但是这样做同样带来一个问题 由于map集合可以看作是一个数组加链表的结合(数组的每一个元素是链表)
根据源码的分析  可以理解为它是一个字典(hash值运算相同的看成是页码  具体hash值对应的是一个个具体对象)
    例如 x & 3 操作    (3,4,5,6,7..)
    0        1        2        3    假设这一行代表 x & 3的值
    --------------------------
    4        5        6        3
    |        |        |        |
    8        9        10        7
    |        |        |        |
    ..        ..        ..        ..
但hash值完全一致的时候 都是往同一个Node添加 这构成了一个桶式结构 
    0
    |
    element1
    |
    element2
    |
    ...
那么比较的时候需要遍历这个结构的所有元素 时间复杂度O(n)    可以在重写equals方法加一个打印作为验证

hashCode返回值的优化方案

作对象唯一的操作就是为了让定义的内容相同的对象不重复出现 但固定hash值又面临全部内容比较的问题
利用数组快速查询索引的优势可以将hash值与内容作对应操作
    如果成员变量是引用类型 取成员的hashCode()返回值 如果是数值类型则直接使用 然后作加法操作
当然可能出现这种情况    
    假设 name.hashCode()=30 而 age=20
        name.hashCode()=20 而 age=30
当数据量大的时候为了细分提升效率不妨将数值类型数据作乘运算
当然可以使用编译器自动生成 hashCode与equals

linkedHashSet概叙及使用

链表结构的set集合
哈希表保证元素的唯一性
链表保证元素有序(存储和取出一致)
相当于是将HashSet按集合长度一定规则排序hash值改为了链表执行
linkedHashSet类继承自HashSet   它自身只有几个构造方法与一个spliterator()方法

TreeSet概叙与使用

基于 TreeMap 的 NavigableSet 实现。使用元素的自然顺序对元素进行排序，或者根据创建 set 时提供的Comparator进行排序，具体取决于使用的构造方法

构造方法以及特点

构造方法:
排序方式有两种
    A: 自然排序
    B: 比较器排序
TreeSet集合的特点:    排序及唯一

源码解析

TreeSet

集合是用NavigableMap存放元素的,而其使用的是TreeMap作了实现
从以上源码可以看出 TreeMap是一个自平衡的二叉树结构 小的元素一直都在左边而大的都在右边 又使用比较器保证了元素的唯一性
对比LinkedHashSet  搜索起来更快一些  每次都可以减少一半的搜索量 而链表需要迭代整个结构
如果是从小到大遍历也很方便 只需要递归 先打印小的元素就可以了

使用TreeSet进行排序的几种方式以及选择

1.将要进行排序的类实现Comparable接口

这样做可以直接在类里面进行排序,但是也有一个弊端.如果我的排序规则进行改变那么类中对应的方法就要作修改
这种做法可能导致一个结果,那就是其实使用这些功能的类也会受到影响

2.创建一个实现了Comparator接口的类作为TreeSet的构造参数

这种做法可以做到排序改变而尽量少的修改相应类的代码
一般适用于比较器使用较多的情况

3.使用匿名内部类实现

这种做法使用于比较器使用只有一次的情况，不必要特意的创建一个类。节约空间
但是如果有重复代码则应该将它们抽取出来,使用创建一个类的策略
